**Концепция проектирования и разработки мобильного приложения для обучения детей написанию стихов**

1.  **Введение**

Настоящая концепция разработана в рамках занятий по проектному практикуму. Целью концепции является разработка мобильного приложения, позволяющая детям научиться писать стихи.

Предметная область – геймификация обучения – охватывает потребность людей к познанию чего-то нового, развитие творческих навыков и популизации стихотворного творчества.

1.  **Описание проблемы, цели, ограничений и приоритетных клиентских путей**

- **Проблемы:**
- Отсутствие приложение, помогающие писать стихи;
- Желание людей обучаться новому;
- Желание родителей, чтоб игры были полезными.
- **Цели:**
- Геймифицировать процесс обучения написания стихов;
- Сделать обучением понятным и интересным;
- Популизировать поэзию.
- **Ограничения:**
- Необходимость строгого соблюдения требований Федерального закона №152-ФЗ «О персональных данных», так как система обрабатывает медицинские и биометрические данные клиентов;
- Соблюдение этических норм;
- Использования понятного языка для детей;
- Ограниченный временные рамки, определяемые учебным планом;
- Понятный на интуитивном уровне интерфейс.
- Приоритетные клиентские пути:

1.  Ребёнок> регистрация>начало обучения>уведомления-напоминалки>прохождение уроков>освоение основ
2.  Родитель>скачивание приложение>занятой ребенок
3.  Пользователь>регистрация>начало обучения>уведомления-напоминалки>прохождение уроков>освоение нового навыка
4.  **Общее описание архитектуры**

Архитектура системы строится по модульному принципу, ориентированному на предметную область «Геймификация обучения»:

**Функциональная архитектура:** клиентоцентричные процессы, сгруппированные по этапам прохождения игры(начать обучение,отправить задание, закончить обучение).

**ИТ-архитектура:** бэкенд написанный на Kothlin, фронтенд написанный на Java, html.

**Данные:** единая модель данных, включающая сущности «Пользователь», «Уроки», «Задание», «Результат.

**Инфраструктура:** разработка на Android Studio

**Диаграммы:**

Контекстная диаграмма

Основные пользователи:

- Ребёнок – активно использует приложение для прохождения уроков.
- Родитель – заинтересован в полезном досуге ребёнка.

Внешние системы:

Сервис PUSH-уведомлений – напоминает о занятиях.

Контейнерная диаграмма

- **Мобильное приложение:**
    - Интерфейс для ребёнка.
    - Реализован на Java и HTML, разработан в Android Studio.
- **Бэкенд-система:**
    - Обрабатывает логику уроков, заданий, результатов.
    - Написана на Kotlin.
    - Обеспечивает REST API для коммуникации.
- **База данных:**
    - Хранит данные о пользователях, уроках, заданиях и результатах.

Компонентная диаграмма

- **Модуль уроков:** Управление структурой уроков, прогрессом пользователя.
- **Модуль заданий:** Приём и проверка заданий, отправленных ребёнком.
- **Модуль пользователя:** Регистрация, управление профилем.
- **Модуль безопасности:** Аутентификация, авторизация по ролям (RBAC), шифрование.
- **Модуль уведомлений:** Интеграция с PUSH-сервисами для напоминаний.

1.  **Связность и интеграция**

Концепция предусматривает возможность интеграции внутренними системами:

- **PUSH-уведомления**

Все интеграции реализуются через стандартизированные REST API с соблюдением требований безопасности.

1.  **Принципы проектирования**

Проектирование архитектуры осуществляется на основе следующих принципов:

- **Клиентоцентричность:** интерфейсы и процессы проектируются исходя из удобства клиентов и сотрудников центра.
- **Масштабируемость и адаптируемость:** архитектура позволяет добавлять новые модули (например, модуль рейтинга без перестройки ядра системы.

- **Простота и удобство использования:** интуитивно понятный пользовательский интерфейс, помогающий увеличить вовлеченность в процесс

- **Технологическая независимость**: приложение самостоятельно

**6) Информационная безопасность**

Безопасность обеспечивается на всех уровнях:

- **Персональные данные** обрабатываются в строгом соответствии с ФЗ-152.
- **Сервисы** разрабатываются по методологии безопасного программного обеспечения
- **Аутентификация и авторизация:** строгое разграничение прав доступа на основе ролей (RBAC)
- **Криптографическая защита:** обязательное шифрование передаваемых (HTTPS/TLS) и хранимых данных.

Предварительно системе присваивается высокий уровень защищенности. Окончательная оценка будет определена по результатам моделирования угроз и аудита.

**7\. Экономическая эффективность**

Внедрение мобильного приложения обеспечивает следующие экономические эффекты:

- **Снижение операционных затрат** за счет автоматизации объяснения материала и проверок заданий;
- **Снижение времени на поиск пользователей**
- Повышение удовлетворённости за счёт геймификации
